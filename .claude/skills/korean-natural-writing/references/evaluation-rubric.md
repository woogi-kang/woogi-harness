# 한국어 자연화 평가표

문체 개선과 의미 보존을 한 점수로 뭉개지 않는다. hard gate를 먼저 적용하고, 통과한 후보만 문체 축으로 평가한다.

## 1. Meaning-preservation hard gate

다음 항목은 하나라도 실패하면 후보를 탈락시킨다.

| 항목 | 확인 질문 |
|---|---|
| 명제 | 원문이 주장한 사실, 조건, 원인, 결과가 그대로인가? |
| 극성·강도 | 부정, 가능성, 의무, 추측, 확신의 강도가 바뀌지 않았는가? |
| 주체 | 누가 판단하고 행동하고 책임지는지가 바뀌지 않았는가? |
| 고정 요소 | 고유명사, 수치, 날짜, 단위, 링크, 인용, 코드 식별자를 보존했는가? |
| 근거 | 원문에 없는 통계, 후기, 성과, 감정, 장면을 만들지 않았는가? |
| 요청 | 분량, 형식, CTA, 독자, 금지 표현 제약을 지켰는가? |
| 의미 역할 | 기준과 결과, 가능과 의무, 행동과 해석처럼 핵심 명사의 역할이 바뀌지 않았는가? |
| 장면 재료 | 원문에 없던 시간 경과, 감각, 움직임, 대화 방식이 생기지 않았는가? |
| 화행 | 안내, 요청, 권고, 명령의 강도가 바뀌지 않았는가? |

자동 토큰 대조는 고정 요소의 일부만 확인한다. 최종 판정 전에 원문과 문장 단위로 의미를 대조한다.
평가 사례에 `required_meaning`이 있으면 항목마다 후보의 실제 근거 구간을 인용한다. task, genre, register도 각각 통과 여부와 실제 구간을 기록하며, 하나라도 빠지면 hard gate를 통과시키지 않는다.

## 2. Pointwise 문체 평가

후보를 서로 비교하기 전에 각 후보를 독립적으로 1–5점으로 평가한다. 다섯 점수 축마다 문제 구간 또는 유지 근거를 후보에서 직접 인용한다.

### 자연스러움

- `5`: 해당 관계와 매체에서 원어민이 실제로 쓸 법하며 번역 흔적이 없다.
- `4`: 대체로 자연스럽고 국소 수정만 필요하다.
- `3`: 뜻은 통하지만 문장 구조나 어휘 선택에 기계적 흔적이 남는다.
- `2`: 여러 문장에서 직역, 모호한 주체, 부자연스러운 호응이 읽기를 막는다.
- `1`: 한국어 문장으로 안정적으로 이해하기 어렵다.

### 장르·관계 적합성

- `5`: 장르, 매체, 독자, 화자 관계, 높임과 종결 문체가 일관된다.
- `4`: 역할은 분명하며 사소한 톤 편차만 있다.
- `3`: 무난하지만 다른 장르에도 그대로 쓸 수 있는 중립 문체다.
- `2`: 카피가 보고서 같거나 실무문이 광고처럼 들리는 등 역할이 어긋난다.
- `1`: 독자와 목적에 맞지 않는다.

### 목소리와 구체성

- `5`: 관찰, 판단, 어휘 선택에서 작성자나 브랜드의 구체적 목소리가 난다.
- `4`: 목소리가 대체로 살아 있고 범용 표현이 드물다.
- `3`: 매끈하지만 누구나 쓸 수 있는 문장이다.
- `2`: 상투어와 추상 평가가 실제 내용보다 많다.
- `1`: 원래 목소리와 핵심 구체성이 사라졌다.

기존 원고를 편집할 때는 새 개성을 만드는 능력이 아니라 원문의 목소리를 얼마나 보존했는지 평가한다. 별도 문체 프로필이 없는 마이크로카피와 실무문에는 개성을 추가하라고 요구하지 않는다.

### 담화와 리듬

- `5`: 정보 순서, 문단 전개, 생략과 연결, 문장 호흡이 독자의 읽기 흐름을 돕는다.
- `4`: 흐름이 자연스럽고 국소 반복만 있다.
- `3`: 논리는 맞지만 같은 구조와 어미가 눈에 띈다.
- `2`: 문단이 반복되거나 인과·지시 관계가 흔들린다.
- `1`: 글 전체의 흐름을 따라가기 어렵다.

리듬은 문장 길이의 다양성만 뜻하지 않는다. 마이크로카피에서는 상태와 행동의 결속, 스토리에서는 장면 시간의 보존, 실무문에서는 조건과 행동의 논리 순서처럼 장르의 기능으로 해석한다.

### 절제와 편집 적정성

- `5`: 필요한 만큼만 고쳐 원문의 장점과 개성을 보존했다.
- `4`: 개선이 분명하고 과수정이 거의 없다.
- `3`: 좋아진 부분과 불필요하게 매끈해진 부분이 함께 있다.
- `2`: 감정, 수사, 단정, 압축을 과도하게 더하거나 뺐다.
- `1`: 다른 글로 바뀌었거나 원문보다 나빠졌다.

## 3. 합격 조건

- hard gate 전부 통과
- 자연스러움, 장르·관계 적합성, 담화와 리듬이 각각 4점 이상
- 나머지 축이 3점 미만이 아님
- 문체 프로필이 있으면 목소리와 구체성이 4점 이상

총점이 높아도 hard gate 실패를 덮지 못한다. 한 후보는 목표 문제를 다른 후보보다 분명히 개선하고 다른 축을 악화시키지 않았을 때만 이긴다. 점수가 같거나 근거가 흔들리면 승자를 만들지 않는다. 최종본이 필요하면 두 후보가 독립적으로 같은 방향으로 고친 패치만 남기고, 논쟁적인 구간에서는 원문 대비 의미 변화가 작은 쪽을 택한다.

## 4. 익명 비교

1. 후보 이름, 사용 스킬, 모델, 반복 횟수, 작성자 메모를 숨기고 `## 완성본` 본문만 비교 파일에 넣는다.
2. 먼저 pointwise 평가를 끝낸다.
3. 원문과 편집 계약을 보며 `A / B / Tie`를 고른다.
4. A와 B 위치를 바꿔 다시 판단한다.
5. 두 판단이 같고 승자가 hard gate를 통과할 때만 자동 선택한다.

순서를 바꿨을 때 승자가 달라지거나, 한 후보가 단지 더 길고 권위적으로 들려 이긴 경우에는 `needs_human_review`로 둔다.
명시된 사용역·의미·형식 계약을 한 후보만 어기면 실질적 차이로 본다. 두 후보가 모두 계약을 충족하고 국소 축 하나의 1점 차이만 있으며 구체적인 독자 효과를 설명할 수 없다면 승자를 만들지 않고 `Tie`로 둔다.

## 5. 비평 형식

```json
{
  "hard_gate": {
    "passed": true,
    "failures": []
  },
  "scores": {
    "naturalness": 4,
    "genre_register_fit": 5,
    "voice_specificity": 4,
    "discourse_rhythm": 4,
    "editing_restraint": 4
  },
  "semantic_deltas": [
    {
      "source_excerpt": "원문의 실제 구간",
      "candidate_excerpt": "후보의 실제 구간",
      "axis": "modality",
      "licensed": false,
      "reason": "가능 안내가 행동 요청으로 강화됨"
    }
  ],
  "findings": [
    {
      "excerpt": "문제 구간",
      "axis": "discourse_rhythm",
      "impact": "독자가 겪는 문제",
      "edit_intent": "실행 가능한 수정 의도"
    }
  ],
  "verdict": "pass",
  "confidence": 0.8
}
```

근거 구간 없이 점수만 주지 않는다. `confidence`는 0과 1 사이 수치로 기록한다. 신뢰도가 낮은 지적만 남으면 자동 수정하지 않는다.
의미 차이가 없으면 `semantic_deltas`를 빈 배열로 둔다. `licensed=false`가 하나라도 있으면 총점과 무관하게 hard gate를 실패시킨다.

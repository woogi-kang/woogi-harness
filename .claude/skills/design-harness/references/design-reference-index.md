# Design Reference Index

Use this as the broad inventory before selecting a detailed reference. Keep it cheap to scan: pick candidates here, then load `design-references.md` or the specific external DESIGN.md only when needed.

Current public source inventory: getdesign.md exposes 73 public DESIGN.md files through its agent-skills index. These are inspiration snapshots, not official design systems.

## How To Use

1. Start with the user's product job and audience.
2. Pick 2-4 candidate references from this index.
3. Choose one primary, one optional secondary, and one anti-reference.
4. Load the exact DESIGN.md only when detailed token/component guidance is needed.
5. Translate grammar into local tokens; do not copy identity.

## Expansion Rule

When adding more references:

- Prefer references with actual DESIGN.md files, real product screenshots, and strong component grammar.
- Add a compact index row first.
- Promote to detailed profile only after it has been used at least twice or fills a distinct design job.
- Keep detailed profiles under `design-references.md`; keep broad inventory here.

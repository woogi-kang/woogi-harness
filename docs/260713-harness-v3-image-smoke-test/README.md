# Harness Runtime v3 Image Smoke Test

Design Read: Reading this as a five-asset harness smoke test for maintainers, in a material editorial workshop, using the asset register, with authored evidence-first variety, avoiding purple AI glow, fake interface evidence, and generic SaaS motifs.

Five dials: `DISTINCTION 8 / MOTION 1 / DENSITY 6 / EVIDENCE 8 / SYSTEMNESS 9`

Borrowed principles:

- Concrete materials and physical evidence instead of abstract AI symbolism.
- One dominant visual idea per asset.
- Exact palette, hierarchy, and output ratio contracts.

Forbidden traits:

- Purple/blue AI glow, glass panels, fake terminals, generic card grids.
- Local prompt suffixes or alternate provider/model fallback.
- Post-generation text overlay or locally claimed model attestation.

Local translation:

- Five different Gongnyang categories exercise typography, product explanation, icon material, cinematic narrative, and presentation composition.
- Every prompt is validated by the exact vendored `image-prompt@2.3.0` validator before generation.
- Generated artifacts record the Codex trusted-host contract rather than claiming a locally observed model.

Artifacts and final QA are recorded in `image-generation-manifest.json` and `test-results.md` after generation.

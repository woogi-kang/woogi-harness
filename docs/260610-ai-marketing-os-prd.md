# PRD: Account Intelligence Marketing OS

## 0. 자동 게시는 어떻게 가능한가

소셜 플랫폼 자동 게시의 핵심은 비공식 자동화가 아니라 각 플랫폼의 공식 API를 쓰는 것이다. 사용자가 계정을 연결하면 OAuth로 권한을 받고, 서버는 승인된 토큰으로 플랫폼별 게시 API를 호출한다. 이후 예약 시간이 되면 백엔드 작업 큐가 게시 요청을 실행하고, 성공 여부와 게시물 ID를 저장한다.

기본 흐름은 다음과 같다.

1. 사용자가 Instagram, Facebook Page, LinkedIn, Threads, TikTok, YouTube, X 계정을 연결한다.
2. 서비스가 OAuth 동의 화면에서 게시 권한을 요청한다.
3. 토큰은 암호화해서 저장한다.
4. 사용자가 콘텐츠를 만들고 승인한다.
5. 시스템이 플랫폼별 규격에 맞춰 문구, 이미지, 영상, 해시태그를 변환한다.
6. 예약 시간이 되면 작업 큐가 공식 API로 게시한다.
7. 게시 결과, 외부 게시물 ID, 실패 사유, 재시도 기록을 저장한다.
8. 이후 조회 가능한 플랫폼에서는 좋아요, 댓글, 공유, 조회수 같은 지표를 수집한다.

중요한 점은 “모든 개인 계정에 마음대로 자동 게시”가 되는 것은 아니라는 점이다. 플랫폼마다 가능한 범위가 다르다.

- Instagram은 Instagram Graph API를 통해 이미지, 영상, 릴스, 캐러셀 게시를 지원하지만 보통 Professional 계정과 Meta 앱 권한이 필요하다.
- Facebook은 개인 프로필보다 Page 게시가 현실적이다. Pages API로 Page 콘텐츠를 만들고 관리할 수 있다.
- Threads는 Threads API로 텍스트, 이미지, 영상, 캐러셀 게시가 가능하다.
- LinkedIn은 개인 프로필 또는 회사 페이지 게시가 가능하지만, 제품 권한과 심사가 필요할 수 있다.
- TikTok은 Content Posting API로 직접 게시 또는 업로드 흐름을 제공하지만, 앱 등록, 도메인 검증, 사용자 승인, UX 가이드 준수가 필요하다.
- YouTube는 YouTube Data API의 업로드 API로 영상을 올릴 수 있다.
- X는 X API의 게시 엔드포인트로 글을 만들 수 있지만, 현재 정책상 접근 플랜과 비용을 고려해야 한다.

### 자동 게시 비용과 제약 검토

자동 게시 자체가 항상 유료인 것은 아니다. 하지만 “무료로 마음껏 게시”도 아니다. 비용은 크게 세 가지로 나뉜다.

1. 플랫폼 API 직접 비용
2. AI 생성 비용
3. 개발, 심사, 운영 비용

플랫폼별 검토 결과는 다음과 같다.

| 플랫폼 | 직접 API 비용 | 핵심 제약 | MVP 판단 |
|---|---:|---|---|
| Instagram / Facebook / Threads | 공식 문서상 게시 API 자체의 공개 per-post 과금은 확인되지 않음 | Meta 앱, 권한, App Review, Rate Limit, Professional 계정 필요 | 우선 검토. 다만 심사 전에는 내부 테스트 또는 수동 게시 대체 |
| LinkedIn | LinkedIn 약관상 현재 API 사용료는 무료이나 향후 과금 가능 | `w_member_social`, `w_organization_social` 권한 필요. 약관상 자동화 사용 해석 리스크 있음 | 개인 브랜딩 글 생성은 우선 지원. 자동 게시 MVP는 보류 |
| TikTok | 공식 문서상 Content Posting API 자체의 공개 per-post 과금은 확인되지 않음 | 앱 등록, 도메인/URL 검증, `video.publish` 승인, 감사 전 콘텐츠 공개 범위 제한 | 영상 파이프라인 이후 2차 |
| YouTube | 현금 과금보다 quota 제약이 핵심 | 기본 quota와 업로드 quota가 제한됨. 대량 업로드는 quota audit 필요 | Shorts/영상은 2차 |
| X | 유료. 2026-06 기준 pay-per-use 크레딧 모델 | 일반 게시 생성은 요청당 과금. URL 포함 게시 비용이 더 높음 | MVP 자동 게시 우선순위에서 제외. 수동 게시 또는 예산 승인 후 진행 |
| 통합 소셜 API | 유료 | Ayrshare 같은 서비스는 월 구독/프로필 기반 과금 | 빠른 검증용 옵션. 장기적으로 직접 연동과 비교 |

2026-06-10 기준으로 가장 현실적인 비용 판단은 다음과 같다.

- 내부 운영 도구로 시작하면 플랫폼 API 비용보다 AI 생성 비용과 개발 시간이 더 크다.
- X 자동 게시는 명확한 유료 항목이므로 기본 기능에서 빼야 한다.
- LinkedIn은 비용보다 정책 리스크가 더 크다. 글 생성과 수동 게시를 먼저 지원한다.
- Meta 계열은 비용보다 App Review와 계정 조건이 병목이다.
- TikTok과 YouTube는 영상 제작, 파일 저장, 업로드 상태 관리까지 필요하므로 MVP 뒤로 미룬다.
- 빠르게 검증하고 싶다면 Ayrshare, Buffer 같은 외부 게시 도구를 붙이는 방법이 있다. 단, 이 경우 “우리 제품의 핵심 기술”은 약해지고 월 구독 비용이 생긴다.

따라서 이 제품은 다음 원칙을 가져야 한다.

- 비밀번호를 저장하거나 브라우저를 몰래 조작하지 않는다.
- 공식 API와 OAuth만 사용한다.
- 플랫폼 심사를 통과하지 못한 채널은 “복사하기”, “알림”, “수동 게시 체크리스트”로 대체한다.
- 사용자가 최종 승인한 콘텐츠만 게시한다.
- 각 플랫폼의 게시 제한, 미디어 규격, 광고/협찬 표시 정책을 검증한다.
- 채널별 월 비용 한도를 설정하고, 유료 API는 기본 비활성화한다.
- 게시 API가 정책상 불확실한 경우 생성/예약/알림까지만 제공한다.

## 1. Summary

이 문서는 현재 운영 중인 서비스들과 개인 SNS 브랜딩을 위한 Account Intelligence Marketing OS의 제품 요구사항을 정의한다.

목표는 계정별 도메인 지식, 기존 콘텐츠 맥락, 플랫폼별 hook 학습을 기반으로 콘텐츠 전략, 블로그 작성, SNS 제작, 승인, 성과 분석을 하나의 흐름으로 연결하는 것이다. WriterRun처럼 자동 게시까지 확장할 수 있지만, 첫 버전은 자동 배포 플랫폼이 아니라 내부 운영용 계정 인텔리전스 도구로 만든다.

## 3. Background

AI로 콘텐츠를 많이 만드는 것은 쉬워졌다. 하지만 많은 콘텐츠가 곧 성과로 이어지지는 않는다. 검색 결과 클릭은 줄고, 사용자는 점점 검색 결과 목록보다 AI 답변을 먼저 본다.

이 변화 때문에 단순 SEO보다 GEO, 즉 AI 답변에 인용될 수 있는 콘텐츠 전략이 중요해지고 있다. 또한 개인 브랜딩과 서비스 마케팅은 꾸준함이 핵심인데, 매번 주제 선정, 글쓰기, 이미지 제작, 게시, 분석을 수동으로 하면 지속하기 어렵다.

우리가 만들 시스템은 “콘텐츠를 대신 많이 만드는 도구”가 아니다. 계정에 맞는 도메인 지식을 계속 축적하고, 기존 콘텐츠의 맥락을 이해하며, 채널별 최신 어투와 hook을 조사해서 배포와 분석까지 반복하는 마케팅 운영 시스템이다.

## 4. Objective

### 목표

운영 중인 서비스와 개인 브랜드를 위한 콘텐츠 마케팅 루프를 체계화한다.

사용자는 매주 다음 일을 할 수 있어야 한다.

- 계정별 도메인 지식과 브랜드 맥락을 계속 업데이트한다.
- 기존 업로드 콘텐츠에서 계정의 말투, 주제, 성과 패턴을 이해한다.
- 플랫폼별 최신 hook, 포맷, 유행 문법을 조사한다.
- 서비스별 브랜드 가이드와 타깃을 관리한다.
- 블로그, LinkedIn, X, Threads, Instagram, YouTube Shorts, TikTok용 콘텐츠를 한 번에 기획한다.
- 각 채널에 맞는 문구와 비주얼을 생성한다.
- 승인한 콘텐츠를 수동 게시하거나, 가능한 채널에서는 예약 게시한다.
- 성과를 보고 다음 콘텐츠 전략에 반영한다.

### Key Results

- 2주 안에 개인 브랜드 또는 운영 서비스 1개를 첫 계정으로 등록한다.
- 2주 안에 첫 계정의 기존 콘텐츠 20개 이상을 분석한다.
- 2주 안에 플랫폼별 hook 후보 30개 이상을 저장한다.
- 8주 안에 3개 서비스와 개인 브랜드 1개를 시스템에 등록한다.
- 8주 안에 계정별 도메인 지식베이스와 기존 콘텐츠 분석 리포트를 만든다.
- 8주 안에 플랫폼별 hook library를 만들고 매주 10개 이상의 신규 hook 후보를 업데이트한다.
- 8주 안에 최소 4개 채널의 콘텐츠 생성 워크플로우를 지원한다.
- 8주 안에 최소 2개 채널은 수동 게시 보조를 지원한다. API 기반 예약 게시는 성공 기준이 아니라 확장 목표다.
- 콘텐츠 1개를 기획부터 예약까지 처리하는 시간을 60분 이하로 줄인다.
- 매주 10개 이상의 게시 후보를 만들고, 최소 5개를 승인 또는 예약한다.
- 12주 동안 채널별 콘텐츠 성과를 누적하고, 상위 20% 콘텐츠 패턴을 리포트한다.
- 월 외부 API/AI/인프라 총비용을 1차 MVP에서는 50달러 이하로 유지한다. 외부 통합 게시 API를 쓰면 별도 승인한다.

## 5. Market Segments

### 1차 사용자: 나 자신

운영 중인 여러 서비스와 개인 브랜딩을 동시에 관리해야 한다. 문제는 시간이 부족하고, 채널마다 문법이 다르며, 꾸준히 게시하기 어렵다는 점이다.

### 2차 사용자: 1인 창업자 / 소규모 팀

마케팅 전담 인력이 없지만 블로그, SNS, 뉴스레터, 숏폼을 운영해야 한다. 브랜드 톤을 유지하면서 콘텐츠를 반복 생산하고 싶다.

### 3차 사용자: 마케팅 실무자

이미 콘텐츠를 만들고 있지만 기획, 제작, 배포, 분석이 흩어져 있다. 반복 업무를 줄이고 성과 분석을 더 빠르게 보고 싶다.

## 6. Value Propositions

### 고객이 얻는 것

- 매번 처음부터 생각하지 않아도 되는 브랜드 기억 장치
- 서비스별 타깃, 톤, 금지 표현, 핵심 메시지 정리
- 블로그와 SNS를 동시에 고려한 콘텐츠 기획
- 플랫폼별 규격에 맞는 문구, 이미지, 해시태그, CTA 생성
- 예약 게시와 성과 분석으로 이어지는 운영 루프
- 개인 브랜딩과 서비스 마케팅을 한 화면에서 관리

### 피할 수 있는 고통

- 콘텐츠 주제를 매번 새로 고민하는 시간
- AI 결과물이 브랜드와 맞지 않는 문제
- 채널마다 다시 쓰고 다시 편집하는 반복 작업
- 게시를 까먹거나 일정이 밀리는 문제
- 성과 데이터를 보고도 다음 액션으로 연결하지 못하는 문제

### 경쟁 제품보다 잘해야 하는 점

- 단순 생성보다 브랜드 맥락을 잘 기억해야 한다.
- 내부 서비스 문서와 실제 운영 맥락을 반영해야 한다.
- SEO뿐 아니라 GEO 관점의 콘텐츠 구조를 제공해야 한다.
- “생성 후 끝”이 아니라 게시와 분석까지 이어져야 한다.
- 개인 브랜딩과 제품 마케팅을 같은 운영 체계에서 다뤄야 한다.

### Non-goals

초기 버전에서 하지 않을 일은 다음과 같다.

- 모든 소셜 플랫폼 자동 게시
- 비공식 스크래핑이나 로그인 세션 자동화
- 대량 AI 영상 생성
- 완전 자동 게시 승인
- 외부 사용자용 SaaS 결제/팀 기능
- 경쟁사 콘텐츠 원문 복제
- 출처 없는 통계와 사실 주장 자동 생성

## 7. Solution

### 7.1 UX / User Flow

#### Flow 0. 계정 인텔리전스 초기화

1. 사용자가 첫 계정을 만든다. 예: 개인 브랜딩, 특정 서비스, 회사 브랜드.
2. 시스템이 계정 목적, 주제, 타깃, 금지 주제, 공개 가능 범위를 받는다.
3. 사용자가 기존 콘텐츠 CSV, URL, 수동 입력 중 하나로 콘텐츠를 넣는다.
4. 시스템이 말투, 주제, hook, CTA, 성과 패턴을 분석한다.
5. 사용자가 참고 계정과 경쟁 계정을 등록한다.
6. 시스템이 초기 hook library와 콘텐츠 기둥을 만든다.

#### Flow A. 브랜드 등록

1. 사용자가 서비스 URL, 소개 문서, 기존 콘텐츠, 타깃 정보를 입력한다.
2. 시스템이 브랜드 정보, 핵심 메시지, 톤앤매너, 금지 표현, 타깃 페르소나를 만든다.
3. 사용자가 초안을 수정하고 승인한다.
4. 승인된 브랜드 가이드는 이후 모든 콘텐츠 생성에 사용된다.

#### Flow B. 콘텐츠 캠페인 생성

1. 사용자가 목표를 고른다. 예: 신규 기능 홍보, 개인 브랜딩, 제품 교육, 사례 공유.
2. 시스템이 1주 또는 1개월 콘텐츠 캘린더를 만든다.
3. 사용자가 주제와 채널을 고른다.
4. 시스템이 블로그, SNS, 숏폼 스크립트, 이미지 프롬프트를 만든다.

#### Flow C. 콘텐츠 편집과 승인

1. 사용자는 생성된 콘텐츠를 한 화면에서 본다.
2. 문장 다듬기, 분량 조절, 스타일 변경, 독자 반응 예측을 실행한다.
3. 필요한 경우 AI 이미지를 생성한다.
4. 사용자가 승인하면 예약 게시 단계로 넘어간다.

#### Flow D. 예약 게시

1. 사용자가 채널별 게시 시간과 계정을 선택한다.
2. 시스템이 플랫폼별 미디어 규격과 글자 수를 검사한다.
3. MVP에서는 복사하기, 게시 체크리스트, 게시 알림을 제공한다.
4. API 연동이 허용된 채널은 예약 시간이 되면 작업 큐가 플랫폼 API로 게시한다.
5. 실패하면 원인을 보여주고 재시도 또는 수동 게시로 전환한다.

#### Flow E. 성과 분석

1. 시스템이 플랫폼별 성과 데이터를 수집한다.
2. 게시물별 조회수, 반응률, 클릭, 저장, 댓글을 보여준다.
3. 성과가 좋은 주제, 형식, 시간대, CTA를 요약한다.
4. 다음 콘텐츠 캘린더에 반영한다.

### 7.2 Key Features

#### 0. 계정 인텔리전스 엔진

AI 마케팅 전문 플랫폼의 핵심은 계정별 기억이다. 같은 제품이라도 계정마다 고객, 말투, 성과 패턴, 금지 표현, 잘 먹히는 각도가 다르다. 따라서 모든 콘텐츠 생성은 계정 인텔리전스를 먼저 참고해야 한다.

계정 인텔리전스는 세 부분으로 나뉜다.

1. 계정 도메인 지식
2. 기존 콘텐츠 맥락
3. 플랫폼별 최신 hook과 포맷

이 엔진은 단순 RAG 저장소가 아니다. 출처, 날짜, 신뢰도, 사용 가능 여부, 마지막 검증일을 함께 저장한다. 오래된 정보는 자동으로 재검토 대상으로 올린다.

#### 0-1. 계정별 도메인 지식 축적

각 계정에 맞는 지식베이스를 정기적으로 업데이트한다.

수집 대상은 다음과 같다.

- 서비스 웹사이트
- 제품 소개 문서
- 가격/기능 페이지
- 블로그와 릴리즈 노트
- 고객 FAQ
- 지원 티켓과 고객 질문
- 세일즈/상담 메모
- 경쟁사 페이지
- 업계 리포트
- 커뮤니티 질문
- 검색 키워드와 AI 검색 질의

저장해야 할 정보는 다음과 같다.

- 핵심 제품 설명
- 주요 고객군
- 고객 문제
- 고객이 쓰는 표현
- 주요 기능과 차별점
- 가격/플랜 정보
- 사용 사례
- 반박해야 할 오해
- 자주 나오는 질문
- 업데이트된 제품 사실
- 출처 URL 또는 파일
- 마지막 확인일

업데이트 주기는 정보 종류마다 다르게 둔다.

| 정보 | 권장 주기 | 이유 |
|---|---:|---|
| 제품 기능 / 가격 | 주 1회 | 콘텐츠 오류가 치명적임 |
| 고객 질문 / 지원 이슈 | 주 1회 | 콘텐츠 주제의 원천 |
| 경쟁사 변화 | 격주 1회 | 포지셔닝 변화 감지 |
| 업계 트렌드 | 월 1회 | 전략 콘텐츠 반영 |
| 계정 브랜드 가이드 | 변경 시 + 월 1회 | 말투와 메시지 일관성 유지 |

필수 안전장치는 다음과 같다.

- 출처 없는 사실은 “검증 필요”로 표시한다.
- 가격, 법률, 정책, 고객 수 같은 민감 정보는 자동 게시 전 재확인한다.
- 제품 로드맵, 내부 메모, 비공개 고객 정보는 공개 콘텐츠에 직접 쓰지 않는다.
- 서로 충돌하는 정보가 있으면 콘텐츠 생성에 쓰지 않고 검토 큐로 보낸다.

#### 0-2. 기존 업로드 콘텐츠 맥락 이해

계정에 이미 올라간 콘텐츠는 가장 중요한 학습 데이터다. 시스템은 기존 게시물을 가져와 계정의 “콘텐츠 DNA”를 만든다.

수집 대상은 다음과 같다.

- 게시 날짜와 플랫폼
- 본문, 캡션, 제목
- 이미지/영상 썸네일
- 영상 자막 또는 음성 전사
- 해시태그
- 댓글과 반응
- 조회수, 좋아요, 저장, 공유, 클릭
- 게시 주제
- 사용한 hook
- CTA
- 콘텐츠 포맷

분석 결과는 다음과 같아야 한다.

- 계정의 대표 어투
- 자주 쓰는 문장 구조
- 반복되는 콘텐츠 기둥
- 많이 반응한 주제
- 반응이 약한 주제
- 잘 먹힌 hook 유형
- 피해야 할 표현
- 댓글에서 반복되는 질문
- 재활용 가능한 과거 콘텐츠
- 중복 생성 위험이 있는 주제

이 기능은 새 콘텐츠 생성 전에 다음 질문에 답해야 한다.

- 이 주제는 전에 말한 적이 있는가?
- 전에 말했을 때 성과는 어땠는가?
- 이번에는 어떤 각도로 바꿔야 하는가?
- 이 계정답게 말하려면 어떤 표현을 써야 하는가?
- 플랫폼별로 어떤 포맷으로 재가공할 수 있는가?

#### 0-3. 플랫폼별 최신 hook 조사 엔진

Hook 조사는 수동 감각에 맡기면 금방 낡는다. 시스템은 매주 플랫폼별로 최신 hook과 포맷을 조사하고, 계정에 맞는 형태로 변환해야 한다.

조사 대상은 네 가지다.

1. 내 계정의 과거 성과
2. 경쟁사와 참고 계정의 최근 콘텐츠
3. 플랫폼 공식 트렌드 도구와 리포트
4. 업계 커뮤니티와 실시간 이슈

플랫폼별 hook은 텍스트만 보지 않는다.

| 플랫폼 | 봐야 할 hook |
|---|---|
| LinkedIn | 첫 1~3줄, See more 전 문장, 문제 제기, 데이터, 개인 경험 |
| X | 첫 문장, 짧은 주장, 스레드 첫 포스트, CTA, 대화 유도 |
| Threads | 대화 시작 문장, 질문, 공감, 실시간 반응, 가벼운 의견 |
| Instagram Reels | 첫 1~3초 영상, 화면 텍스트, 얼굴/제품 등장, 사운드 |
| TikTok | 첫 1~3초 영상, 사운드, 자막, 챌린지/밈 구조, 댓글 기반 후속 콘텐츠 |
| YouTube Shorts | 첫 1~3초 영상, 제목, 반복 시청 구조, 시각 변화, 자막 |
| Blog / GEO | 제목, 첫 문단 정의, 요약 박스, 질문형 H2, 비교표 |

Hook 수집 스키마는 다음과 같다.

| 필드 | 설명 |
|---|---|
| platform | LinkedIn, X, Instagram, TikTok 등 |
| source_url | 원본 링크 |
| source_account | 원본 계정 |
| date_collected | 수집일 |
| content_topic | 주제 |
| content_format | 텍스트, 캐러셀, 릴스, 쇼츠, 스레드 등 |
| opening_text | 첫 문장 또는 화면 텍스트 |
| first_3_sec_transcript | 영상 첫 3초 전사 |
| visual_hook | 얼굴, 제품, 전후 비교, 자막, 움직임 등 |
| hook_type | 가치 약속, 질문, 반전, 실수, 논쟁, 데이터, 개인 경험 등 |
| audience | 누구에게 말하는가 |
| promise | 사용자가 얻는 것 |
| emotion | 호기심, 불안, 공감, 놀람, 욕망 등 |
| proof | 데이터, 사례, 경험, 권위 |
| engagement | 조회, 좋아요, 댓글, 저장, 공유 |
| normalized_score | 계정 규모 대비 성과 |
| risk | 저작권, 과장, 브랜드 부적합, 민감 표현 |

Hook 점수는 다음 기준으로 계산한다.

| 기준 | 비중 |
|---|---:|
| 계정 주제와의 관련성 | 30% |
| 성과 신호 | 25% |
| 최신성 | 20% |
| 플랫폼 적합성 | 15% |
| 변형 가능성 | 10% |

다만 다음 항목은 감점한다.

- 원본과 너무 비슷해 표절 위험이 있음
- 브랜드 톤과 맞지 않음
- 과장, 공포 마케팅, 낚시성이 강함
- 법률/의료/금융처럼 검증이 필요한 주장
- 특정 집단을 공격하거나 민감한 표현이 있음

주간 조사 프로세스는 다음과 같다.

1. 내 계정의 최근 30일 콘텐츠 성과를 가져온다.
2. 플랫폼별 참고 계정 20~50개를 확인한다.
3. TikTok Creative Center, YouTube Culture & Trends, Meta/Instagram Reels 공식 팁, LinkedIn B2B 리포트, X Organic Best Practices 같은 공식 자료를 확인한다.
4. 각 플랫폼에서 최근 7일 또는 30일 안에 반복되는 hook 패턴을 추출한다.
5. LLM으로 hook을 유형별로 클러스터링한다.
6. 원문을 복사하지 않고 계정 도메인에 맞는 변형 template으로 바꾼다.
7. 이번 주 추천 hook 10~20개를 만든다.
8. 실제 게시 후 성과를 hook library에 다시 기록한다.

MVP에서는 자동 크롤링을 무리하게 만들지 않는다. 처음에는 수동 조사 + 반자동 저장으로 시작한다. 플랫폼 약관상 스크래핑이 애매한 곳은 공식 API, 공식 트렌드 도구, 수동 리서치, 사용자가 제공한 링크만 사용한다.

#### 0-4. Hook 조사 프로토콜

Hook 조사는 매주 같은 방식으로 실행되어야 한다. 그래야 개인 감각이 아니라 운영 데이터가 쌓인다.

샘플링 기준은 다음과 같다.

- 내 계정 콘텐츠: 최근 30일 게시물 전체, 또는 최근 20개 게시물
- 참고 계정: 플랫폼별 5~10개
- 경쟁 계정: 제품/시장 기준 5~10개
- 트렌드 소스: 공식 트렌드 도구와 리포트 우선
- 기간: 기본 최근 7일, 느린 B2B 채널은 최근 30일

수집 허용 방식은 다음과 같다.

- 사용자가 직접 제공한 게시물 URL
- 플랫폼 공식 API로 허용된 데이터
- 공식 트렌드 도구에서 수동 확인한 패턴
- 공개 웹페이지의 제한적 수동 리서치
- 사용자가 내보낸 CSV 또는 스프레드시트

금지 방식은 다음과 같다.

- 로그인 세션을 몰래 자동화해서 대량 수집
- 플랫폼 약관을 우회하는 스크래핑
- 원본 문구를 그대로 복사해 재사용
- 개인정보가 포함된 댓글을 그대로 학습 데이터로 저장
- 비공개 계정, 유료 커뮤니티, 사내 자료를 출처 표시 없이 사용

라벨링 규칙은 다음과 같다.

- hook은 원문과 변형안을 분리해서 저장한다.
- 원문은 분석용이며, 게시용 산출물에는 그대로 쓰지 않는다.
- 각 hook에는 `hook_type`, `emotion`, `proof`, `risk`, `platform_fit`, `account_fit`을 반드시 붙인다.
- 성과 숫자가 없으면 `performance_unknown`으로 표시한다.
- 조회수만 높은 hook은 저장하되, 브랜드 적합도 점수가 낮으면 추천에서 제외한다.

주간 hook 리포트 포맷은 다음과 같다.

| 항목 | 설명 |
|---|---|
| 이번 주 반복 패턴 | 여러 계정에서 반복된 opening, 시각 구조, 주제 |
| 계정 적용 후보 | 내 계정에 맞게 바꾼 hook template |
| 위험 후보 | 낚시성, 과장, 표절 위험이 있는 hook |
| 테스트할 5개 | 이번 주 실제 게시에 쓸 hook |
| 지난주 결과 | hook별 조회, 반응, 댓글 품질, 저장/공유 |

#### 0-5. 기존 콘텐츠 입력 방식

MVP에서는 플랫폼 API보다 입력 안정성을 우선한다. 자동 수집은 매력적이지만, 권한과 정책 검토가 오래 걸릴 수 있다.

지원 순서는 다음과 같다.

1. CSV 업로드: 플랫폼, 날짜, 본문, URL, 조회수, 좋아요, 댓글, 공유, 저장, 클릭을 입력한다.
2. URL 수동 입력: 게시물 링크를 넣고 본문과 메타 정보를 사용자가 보완한다.
3. 복사/붙여넣기: 기존 글을 직접 붙여넣는다.
4. 공식 API 수집: 채널별 권한 검토 후 추가한다.

MVP 성공 기준은 “자동 수집”이 아니라 “기존 콘텐츠 맥락이 새 콘텐츠 생성에 반영되는 것”이다.

#### 1. 브랜드 브레인

- 서비스 URL 분석
- 기존 문서 업로드
- 브랜드 소개 자동 생성
- 타깃 페르소나 생성
- 브랜드 톤앤매너 생성
- 금지 표현과 선호 표현 관리
- 채널별 보이스 가이드
- 개인 브랜딩용 프로필 가이드

#### 2. 콘텐츠 전략 엔진

- 캠페인 목표 선택
- 주간 / 월간 콘텐츠 캘린더 생성
- 서비스별 콘텐츠 기둥 설정
- 개인 브랜딩 주제 클러스터 설정
- SEO / GEO 키워드와 질문 생성
- 콘텐츠 우선순위 추천

#### 3. 블로그 / GEO 콘텐츠 제작

- 글 주제 제안
- 콘텐츠 브리프 생성
- 본문 초안 생성
- FAQ와 구조화된 답변 생성
- AI 답변에 인용되기 좋은 요약 블록 생성
- 제목, 메타 설명, OG 문구 생성
- 신뢰 근거와 출처 입력 영역
- 독자 반응 예측
- 문장 교정, 톤 변경, 분량 조절

GEO 콘텐츠의 필수 산출물은 다음과 같다.

- 40~60단어 안팎의 직접 답변 블록
- 질문형 H2/H3 구조
- FAQ 3~7개
- 비교표 또는 체크리스트
- 출처가 붙은 핵심 주장
- 마지막 업데이트 날짜
- 작성자 또는 계정 전문성 설명
- 메타 제목, 메타 설명, OG 문구
- schema 후보: `Article`, `BlogPosting`, `FAQPage`, `HowTo`, `Product`
- AI 답변에 인용되기 좋은 요약 박스

Acceptance criteria:

- 출처 없는 통계나 수치 주장을 생성하지 않는다.
- 가격, 기능, 정책 정보는 knowledge item의 최신 검증일을 확인한다.
- “무엇인가”, “어떻게 하는가”, “비교”, “대안” 질문에 바로 답하는 블록이 있어야 한다.
- 문서 안에 사람이 읽는 글과 AI가 추출하기 쉬운 구조가 함께 있어야 한다.
- 기존 콘텐츠와 70% 이상 유사한 제목/구조는 중복 위험으로 표시한다.

#### 4. SNS 콘텐츠 제작

- LinkedIn 긴 글 버전
- X 짧은 글 / 스레드 버전
- Threads 대화형 버전
- Instagram 카드뉴스 / 캡션 버전
- TikTok / Shorts 스크립트 버전
- 해시태그 추천
- CTA 추천
- 채널별 글자 수와 미디어 규격 검사

#### 5. AI 스튜디오

- 제품 이미지 기반 광고컷 생성
- 브랜드 무드에 맞는 썸네일 생성
- 카드뉴스 이미지 생성
- 숏폼 장면 구성안 생성
- 개인 브랜딩 프로필 이미지 스타일 가이드
- UGC 스타일 스크립트와 장면 프롬프트 생성

MVP에서는 고급 영상 생성보다 이미지, 썸네일, 숏폼 스크립트부터 시작한다.

#### 6. 예약 게시 / 배포

- 계정 OAuth 연결
- 채널별 게시 권한 상태 확인
- 예약 캘린더
- 승인 후 게시
- 플랫폼별 게시 어댑터
- 실패 재시도
- 수동 게시 대체 플로우
- 게시물 외부 URL 저장

MVP 우선순위는 수동 게시 보조, Threads, Facebook Page, Instagram Professional 계정이다. LinkedIn은 글 생성과 수동 게시를 먼저 지원하고, 자동 게시는 정책 검토 후 진행한다. X는 유료 API이므로 기본 MVP에서 제외한다. TikTok과 YouTube는 영상 제작 파이프라인이 필요하므로 2차로 둔다.

#### 7. 성과 분석

- 채널별 게시 현황
- 게시물별 반응 지표
- 클릭 추적용 UTM 생성
- 캠페인별 성과 비교
- 좋은 콘텐츠 패턴 요약
- 다음 주 콘텐츠 추천
- 골든타임 분석

#### 8. 운영 안전장치

- 게시 전 인간 승인
- 민감 표현 검사
- 저작권 위험 체크
- 협찬 / 광고 표시 체크
- 개인정보 포함 여부 검사
- 토큰 암호화 저장
- 계정별 권한 분리
- 게시 로그와 감사 로그

Acceptance criteria:

- 모든 게시물은 `draft`, `ready_for_review`, `approved`, `scheduled`, `published`, `failed`, `cancelled` 상태 중 하나를 가진다.
- 자동 게시 또는 게시 알림은 `approved` 상태에서만 실행된다.
- OAuth refresh token은 암호화 저장하고, 계정 연결 해제 시 즉시 폐기한다.
- 게시 실패는 최대 3회까지만 자동 재시도한다.
- 실패 사유와 외부 API 응답 코드는 감사 로그에 남긴다.
- 개인정보, 고객명, 내부 로드맵, 비공개 매출 정보는 기본적으로 `never_publish` 등급으로 분류한다.
- 유료 API를 호출하는 작업은 계정별 월 비용 한도를 넘기면 중단된다.

### 7.3 Cost Model

#### 직접 구축 시 비용

직접 구축은 월 구독비를 줄일 수 있지만 개발 시간이 많이 든다. 비용은 다음 항목에서 발생한다.

- LLM 텍스트 생성 비용
- AI 이미지 생성 비용
- 영상 생성 또는 영상 편집 비용
- 파일 저장소 비용
- 작업 큐와 서버 비용
- 도메인, DB, 로그, 모니터링 비용
- 소셜 API 심사 대응 시간
- 실패 게시 재처리와 고객 지원 시간

내부 운영 MVP에서는 월 총액 50달러를 넘지 않는 것을 기준으로 한다.

| 항목 | MVP 예산 가드레일 |
|---|---:|
| 텍스트 생성 | 월 10~20달러 이내 |
| 이미지 생성 | 기본 제외. 필요 시 월 10달러 이내 |
| 영상 생성 | MVP 제외 |
| X API | 기본 제외 |
| 통합 소셜 API | 기본 제외. 필요 시 별도 승인 |
| 인프라 | 월 10~20달러 이내 |
| 예비 비용 | 월 10달러 이내 |

#### 통합 소셜 API 사용 시 비용

직접 연동 대신 Ayrshare 같은 통합 소셜 API를 쓰면 개발 속도는 빨라진다. 대신 월 구독 비용이 생긴다. 2026-06 기준 Ayrshare는 자체 브랜드 관리용 Premium이 월 149달러, MVP/베타용 Launch가 월 299달러, Business가 월 599달러부터 시작한다.

Buffer 같은 스케줄링 도구를 쓰는 방법도 있다. Buffer는 무료 플랜에서 제한된 채널과 예약 수를 제공하고, 유료 플랜은 채널 단위 과금 구조다. 이 방식은 제품 검증에는 빠르지만, 우리가 만들 시스템의 핵심 기능을 외부 도구에 의존하게 된다.

#### 추천 비용 전략

1. V0에서는 자동 게시를 만들지 않는다. 계정 인텔리전스, 생성, 승인, 복사하기, 게시 체크리스트를 만든다.
2. V1에서는 자동 게시보다 지식 업데이트, 기존 콘텐츠 분석, hook library를 안정화한다.
3. V2에서 수동 게시 운영과 성과 학습 루프를 만든다.
4. V3에서 무료 또는 저비용으로 가능한 채널부터 직접 연동한다.
5. X는 예산 승인 전까지 생성/복사만 지원한다.
6. LinkedIn은 정책 검토 전까지 자동 게시를 보류한다.
7. 외부 통합 API는 빠른 검증이 필요할 때만 붙인다.
8. AI 이미지와 영상은 사용량 제한, 캐싱, 재사용을 기본으로 설계한다.

### 7.5 Assumptions

- 운영 중인 서비스들의 URL과 기본 소개 문서를 확보할 수 있다.
- 개인 브랜딩의 핵심 주제와 피하고 싶은 이미지를 정의할 수 있다.
- Instagram은 Professional 계정으로 전환할 수 있다.
- Meta, TikTok, LinkedIn 등 필요한 플랫폼 심사를 받을 수 있다.
- 모든 채널을 한 번에 자동 게시하지 않아도 MVP 가치는 있다.
- 첫 버전은 완전 자동화보다 “승인 후 자동 예약”이 더 안전하다.
- AI 이미지와 영상 품질은 채널별로 검수 과정이 필요하다.
- X 자동 게시는 유료이므로 초기 핵심 기능이 아니다.
- LinkedIn 자동 게시는 기술 가능성과 별개로 약관/권한 검토가 필요하다.

### 7.6 Risks

| 리스크 | 영향 | 대응 |
|---|---|---|
| 계정 지식이 오래됨 | 잘못된 기능/가격/정책을 말함 | 지식별 검증일, 만료일, 검토 큐 운영 |
| 기존 콘텐츠 데이터가 적음 | ContentDNA 품질이 낮음 | 최소 20개 입력 전에는 “초기 학습 중” 표시 |
| hook이 표절처럼 보임 | 브랜드/법적 리스크 | 원문과 변형안을 분리하고 유사도 검사 |
| 플랫폼 API 정책 변화 | 자동 게시 실패 | 수동 게시 체크리스트를 항상 유지 |
| AI가 출처 없는 사실을 만듦 | 신뢰 하락 | 출처 없는 수치/주장은 게시 전 차단 |
| 비용 초과 | 운영 부담 증가 | 계정별 월 한도와 유료 API 기본 비활성화 |
| 민감 정보 유출 | 치명적 신뢰 문제 | 공개 가능 등급과 `never_publish` 차단 |

### 7.7 Validation Plan

MVP는 다음 순서로 검증한다.

1. 계정 1개에 기존 콘텐츠 20개를 입력한다.
2. 시스템이 ContentDNA 리포트를 만든다.
3. 사용자가 리포트의 말투/주제/hook 분석을 5점 척도로 평가한다.
4. hook library에서 10개 변형안을 만든다.
5. 콘텐츠 후보 5개를 생성한다.
6. 사용자가 “계정답다”, “게시 가능하다”, “중복이 아니다”를 평가한다.
7. 1주일 동안 수동 게시한다.
8. 게시 후 성과를 입력하고, 다음 주 추천이 바뀌는지 확인한다.

통과 기준은 다음과 같다.

- ContentDNA 리포트의 계정 이해도 평균 4점 이상
- 콘텐츠 후보 5개 중 3개 이상 게시 가능
- hook 변형안 10개 중 7개 이상 표절 위험 낮음
- 수동 게시 누락 0건
- 다음 주 추천에 실제 성과 데이터가 반영됨

## 8. Release

### V0. 계정 인텔리전스 스파이크

기간: 1~2주

목표는 “이 계정답게 생성되는가”를 확인하는 것이다. 자동 게시, 이미지 생성, 복잡한 대시보드는 만들지 않는다.

- 개인 브랜드 또는 운영 서비스 1개 등록
- 기존 콘텐츠 20개 입력
- 도메인 지식 source 5개 등록
- hook example 30개 저장
- hook variant 10개 생성
- 콘텐츠 DNA 리포트 생성
- 주간 콘텐츠 후보 5개 생성
- 수동 게시 체크리스트 제공

성공 기준:

- 생성된 콘텐츠 5개 중 3개 이상이 “계정답다”고 판단된다.
- 기존 콘텐츠와 중복되는 주제는 경고된다.
- hook variant가 원문을 복사하지 않고 계정 주제에 맞게 변형된다.
- 월 외부 비용이 50달러 이하로 유지된다.

### V1. 계정 인텔리전스 MVP

기간: V0 이후 3~5주

목표는 계정별 지식, 기존 콘텐츠, hook library를 운영 가능한 데이터 구조로 만드는 것이다.

- 계정 3개까지 등록
- CSV / URL / 복사 입력 지원
- KnowledgeSource, KnowledgeItem, ImportedPost, HookExample 저장
- 공개 가능 등급 관리
- 주간 hook 조사 리포트
- 콘텐츠 DNA 자동 업데이트
- 콘텐츠 생성 시 knowledge item과 hook variant 참조
- 승인 큐와 수동 게시 체크리스트

성공 기준:

- 계정별 지식 충돌과 검증 필요 항목이 검토 큐로 올라온다.
- 주간 콘텐츠 후보 10개 이상을 만들 수 있다.
- 사용자는 생성된 콘텐츠마다 참조된 지식과 hook을 확인할 수 있다.
- 자동 게시 없이도 1주일 운영에 쓸 수 있다.

### V2. 콘텐츠 운영 MVP

기간: V1 이후 4~6주

목표는 기획, 생성, 승인, 수동 게시, 성과 기록이 하나의 루프로 도는 것이다.

- 콘텐츠 캘린더
- 블로그 / GEO 초안 생성
- LinkedIn, X, Threads, Instagram용 문구 생성
- 카드뉴스 텍스트 구조 생성
- UTM 생성
- 수동 게시 알림
- 게시 후 성과 수동 입력 또는 CSV 업로드
- hook별 성과 기록
- 다음 주 추천 주제 생성

성공 기준:

- 콘텐츠 1개를 기획부터 게시 준비까지 60분 이하로 처리한다.
- 한 주에 게시 후보 10개, 실제 게시 5개를 운영한다.
- 게시 후 성과가 hook library와 ContentDNA에 반영된다.

### V3. 자동 게시 / WriterRun형 확장

기간: V2 이후 6~10주

목표는 검증된 콘텐츠 운영 루프에 자동 게시와 시각 자산 생성을 붙이는 것이다.

- OAuth 계정 연결
- Threads, Facebook Page, Instagram Professional 계정 중 가능한 채널 우선 연동
- 예약 캘린더
- 게시 실패 처리
- 카드뉴스 이미지 생성
- 블로그 GEO 체크리스트
- 콘텐츠 진단
- 독자 반응 예측
- 채널별 성과 대시보드

성공 기준:

- 승인한 콘텐츠가 예약 시간에 실제 게시되거나, 수동 게시 알림으로 누락 없이 처리된다.
- 게시 실패 원인이 사용자에게 설명된다.
- 자동 게시가 없어도 생성/승인/성과 루프는 유지된다.

### V4. 외부 사용자용 SaaS 후보

기간: V3 검증 이후

- 팀 권한
- 여러 브랜드 관리
- 결제
- 템플릿 마켓
- 더 많은 플랫폼 연동
- 앱 심사와 운영 정책 정비

성공 기준: 내부 운영 도구를 넘어 다른 1인 창업자나 소규모 팀도 쓸 수 있다.

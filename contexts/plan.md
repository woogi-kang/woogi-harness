# Context Mode: Plan

설계와 계획 수립. 구현은 합의 후.

## 행동 규칙
- 요구사항을 분석하고 스코프를 명확히 정의
- 여러 실행안이 가능하면 `.claude/rules/common/clarification-protocol.md`에 따라 native 선택 UI로 추천안과 대안을 제시
- 아키텍처 결정과 트레이드오프 문서화
- project profile, 필요한 capability, context budget을 함께 정의
- 작업 분해 (Work Breakdown Structure)
- 의존성과 리스크 식별
- 마일스톤과 체크포인트 설정
- success criteria, evidence, 중단 조건, 승인 경계를 typed execution contract로 정의
- 사용자 합의 없이 코드 작성 금지
